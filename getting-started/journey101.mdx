## Client interactions

The lesson below will dive into basic client interactions. It demonstrates how to use the following journey steps:

1. [Get Information from Client](/guides/orchestration/journeys/get_info_from_client/), allows presenting a generic client-side form that collects data
2. [Display Information](/guides/orchestration/journeys/display_information/), allows displaying dynamic information on the client side. These forms are mostly used during journey building and debugging

## Control structures

The lesson below explains how to use the journey control structures:

1. [Generic Condition](/guides/orchestration/journeys/condition/), allows journey branching on dynamic data
2. [Set Temporary Variables](/guides/orchestration/journeys/set_temporary_variables/), allows creating variables that can store data and calculations, to be used throughout the journey
3. [While Loop](/guides/orchestration/journeys/while_loop/), allows wrapping a specific part of the journey as a loop, with variables and exit condition

## Passwords: registration & authentication

The lesson below introduces authentications and implements the password authentication flow.

1. [Register Password](/guides/orchestration/journeys/register_password/), registers a password, creating a user record if needed
2. [Login Form](/guides/orchestration/journeys/login_form/), presents a login form. This is a fundamental step for collecting user credentials, irrespective of authenticator type
3. [Password Authentication](/guides/orchestration/journeys/authenticate_password/), authenticates a user with a password

## TOTP: registration & authentication

The lesson below extends authentications, introducing the TOTP (Time-based One-Time Passwords):

1. [Register TOTP](/guides/orchestration/journeys/register_totp/), registers a TOTP authenticator with an authenticator app
